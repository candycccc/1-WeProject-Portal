Swipe actions in the browser — two approaches

Open this page on a phone or any touch device to try it. On desktop, Variant 1 also works with mouse drag.

Variant 1 · Pointer Events (JS — recommended for production)
How it works: listen to pointerdown / pointermove / pointerup, track horizontal movement with transform: translateX() so the row follows the finger. On release, if it passes the threshold the row snaps open; otherwise it springs back.
Key details: touch-action: pan-y keeps vertical scrolling working; a direction lock (horizontal must exceed vertical before the gesture takes over) prevents scroll/swipe conflicts; only one row can be open at a time; mouse drag is supported, so it's fully testable on desktop.
Variant 2 · CSS scroll-snap (zero JS)
How it works: each row is a horizontal scroll container with the action buttons laid out behind the row face. scroll-snap-type: x mandatory makes it snap open/closed natively.
Pros: no JavaScript at all, native browser momentum, and horizontal trackpad swipes work on desktop.
Cons: low control — no callback to know when a row is open, no way to enforce "only one row open at a time", and Windows mouse users can't drag (they'd need shift+wheel). Good for prototyping; use Variant 1 in production.
